## 🎯 **OBJETIVOS FUNCIONAIS**

### **Objetivo Principal**
Criar um sistema onde o advogado:
1. Digita o número do processo (CNJ ou formato curto, ex: ADI-5643)
2. Clica em "Consultar Processo"
3. O sistema busca em MÚLTIPLOS TRIBUNAIS simultaneamente (STF + STJ)
4. O sistema detecta o tribunal correto
5. Clica no botão "Adicionar ao Monitoramento"
6. O sistema verifica automaticamente a cada 60 minutos
7. Se houver **NOVOS ANDAMENTOS**, envia ALERTA para o Dashboard
8. O advogado pode ver a lista de processos monitorados
9. Clicar em "Ver Andamentos" para ver as movimentações na página oficial

## 📁 **MODELOS DE DADOS**

### **Modelo: ProcessoJudicial**

```prisma
model ProcessoJudicial {
  id               String            @id @default(uuid())
  numeroCnj        String            @unique     // Número CNJ (formato: 0000000-00.0000.0.00.0000)
  numeroTribunal   String?                       // Número no formato curto
  tribunal         String                           // Código do tribunal (STF, STJ, TJSP, etc.)
  orgaoJulgador   String?
  dataDistribuicao String?
  objeto           String?                          // Objeto/Assunto do processo
  classe           String?                          // Classe processual
  area             String?                          // Área (Judicial, Constitucional, etc.)
  valorCausa       Float?
  sigiloProcessual Boolean           @default(false)
  segredoJustica   Boolean           @default(false)
  urlOrigem        String?                          // Link oficial do tribunal
  htmlCompleto      String?                          // HTML bruto completo (para debug)

  // Campos de auditoria
  statusValidacao  String?                          // 'valido', 'parcial', 'invalido', 'mock'
  problemasValidacao String?                        // Descrição dos problemas encontrados
  ultimaAtualizacao DateTime?

  // Campos de timestamp
  createdAt        DateTime          @default(now())
  updatedAt        DateTime          @updatedAt

  // Relações
  partes           Parte[]
  advogados        Advogado[]
  andamentos       Andamento[]
  monitoramentos   Monitoramento[]
}
```

### **Modelo: Andamento**

```prisma
model Andamento {
  id          String   @id @default(uuid())
  processoId   String   // ID do processo (Referência para ProcessoJudicial)
  data        String   // ISO Date (YYYY-MM-DDTHH:MM:SS.sssZ)
  dataFormatada String   // Data formatada (DD/MM/YYYY)
  descricao   String   // Descrição do andamento
  tipo        String?   // Tipo: 'distribuicao', 'andamento', 'movimentacao', etc.

  createdAt   DateTime @default(now())
  processo    ProcessoJudicial @relation("AndamentoToProcesso", fields: [processoId], references: [id], onDelete: Cascade)
}
```

### **Modelo: Monitoramento**

```prisma
model Monitoramento {
  id                  String   @id @default(uuid())
  processoId          String
  usuarioId           String?  // ID do usuário autenticado

  ativo               Boolean  @default(true)
  frequenciaChecagem Int      @default(60)  // Minutos entre checagens (60, 120, 180...)

  // Timestamps
  ultimaChecagem      DateTime?
  proximaChecagem      DateTime?
  lastModifiedHeader String?        // Last-Modified do HTML (para ver se mudou)

  // Controle de novos andamentos
  novosAndamentos     Boolean  @default(false)
  contadorAndamentos  Int      @default(0)

  createdAt           DateTime @default(now())

  // Relação
  processo            ProcessoJudicial @relation("MonitoramentoToProcesso", fields: [processoId], references: [id], onDelete: Cascade)
}
```

## 🛠️ **ARQUITETURA DE ARQUIVOS**

### **Arquivo: `/src/lib/db.ts` (Banco de Dados)**
- **Função:** Cliente Prisma inicializado.
- **Responsável:** Conexão com SQLite (`custom.db`).
- **Comandos:** `db.processoJudicial`, `db.andamento`, `db.monitoramento`.

### **Arquivo: `/src/lib/playwright-scraper.ts` (Playwright Scraper)**
- **Função:** `scrapeAndamentosComPlaywright(numero, tribunal)`
- **Responsável:** Scraper usando Playwright.
- **Comportamento:**
  - Inicializa navegador Chromium (Headless)
  - Navega até a página do tribunal
  - Simula clique na aba "Andamentos"
  - Extrai HTML e tabela de andamentos
  - Retorna dados estruturados.

### **Arquivo: `/src/app/api/processos/scrape/route.ts` (Busca)**
- **Função:** `scrapingUnificado(numero, tribunal)`
- **Responsável:** Endpoint de busca unificada.
- **Comportamento:**
  - Busca em STF e STJ simultaneamente
  - Retorna dados do tribunal correto (se encontrado)

### **Arquivo: `/src/app/api/processos/scrape-playwright/route.ts` (Scraping com Playwright)**
- **Funcionalidade:** Extraição de andamentos usando Playwright.
- **Fallback:** Se Playwright falhar -> Usa Mock.

### **Arquivo: `/src/app/api/processos/monitoramento/route.ts` (Monitoramento)**
- **Funcionalidade: Gerencia watchlist de processos.
- **Endpoints:**
  - `POST /api/processos/monitoramento`
    - `action: 'adicionar'`: Adiciona processo ao monitoramento
    - `action: 'remover': Remove processo do monitoramento
    - `action: 'verificar': Verifica se há novos andamentos agora
  - `action: 'verificar-todos': Verifica todos os processos (Cron Job)
    - `action: 'verificar-todos-agora': Verifica todos os processos agora (manual)

### **Arquivo: `/src/app/monitoramento/page.tsx` (Dashboard)**
- **Funcionalidade:** Interface de monitoramento.
- **Exibe:**
  - Lista de processos monitorados
  - Status: Em monitoramento, Novos eventos, Última checagem, Próxima checagem
  - Alertas visuais (Verde/Piscante) para novos andamentos
  - Botões: "Verificar agora", "Remover", "Acessar página oficial"

## 🧪 **PLAYWRIGHT SCRAPER**

### **Objetivo**
Extrair **ANDAMENTOS COMPLETOS** das páginas dos tribunais brasileiros usando **NAVEGADOR REAL (Chromium Headless)**.

### **Como Funciona**

1. **Inicialização**
   ```typescript
   const browser = await playwright.chromium.launch({
     headless: true,
     args: ['--no-sandbox', '--disable-setuid-sandbox', '--disable-web-security']
   })
   ```

2. **Navegação**
   ```typescript
   await page.goto('https://portal.stf.jus.br/processos/verProcessoAndamento.asp?numero=0000001-21.2005.1.00.0000', {
     waitUntil: 'networkidle'
   })
   ```

3. **Clique na Aba (Simulado)**
   - A aba de "Andamentos" é clicada (simulação de clique)
   - O JavaScript carrega os dados dinâmicos

4. **Extração de Dados**
   ```typescript
   const andamentos = await page.evaluate(() => {
     const rows = document.querySelectorAll('table tbody tr')
     const dados = Array.from(rows).map(row => {
       const cells = row.querySelectorAll('td')
       return {
         data: cells[0]?.textContent?.trim(),
         descricao: cells[1]?.textContent?.trim()
       }
     })
     return dados
   })
   ```

### **Limitações (Honestidade)**
- ⚠️ **Dependências de Sistema**: Requer `libatk`, `libgtk`, `libdrm`, etc. (não estão instalados no ambiente atual do Z.ai, pois não tenho `sudo`).
- ⚠️ **Playwright no Vercel**: Não funciona bem com scraping de navegação (tem timeout de execução).
- ⚠️ **Abas Dinâmicas**: Os andamentos carregados via JavaScript não aparecem no HTML estático inicial, impossível de extrair sem navegador real executando o JS.

## 🤖 **RESOLUÇÃO DE PROBLEMAS**

### **Erro: "Executable doesn't exist" (Chromium)**
```
CAUSA: Falta binário do navegador.
SOLUÇÃO:
1. npx playwright install chromium
2. Se falhar, reinstale o Playwright:
   - Desinstale: `bun remove playwright`
   - Reinstale: `bun add playwright`
```

### **Erro: "Module not found" (Playwright)**
```
CAUSA: Playwright não instalado.
SOLUÇÃO:
bun add playwright
```

### **Erro: "Cannot find module" (Prisma)**
```
CAUSA: Cliente Prisma não foi gerado.
SOLUÇÃO:
bun prisma generate
bun run db:push
```

### **Erro: "Database locked" (SQLite)**
```
CAUSA: Outro processo está usando o banco (ou o servidor Next.js).
SOLUÇÃO:
- Feche o servidor: `pkill -f "bun dev"`
- Reincie: `bun run dev`
```
